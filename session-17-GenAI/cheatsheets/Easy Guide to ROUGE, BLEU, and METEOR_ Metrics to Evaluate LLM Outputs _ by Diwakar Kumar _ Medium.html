When a language model like ChatGPT generates a summary, translates a sentence, or answers a question, how do we know if the output is good?

That‚Äôs where automatic evaluation metrics come in.

The three most common ones are:

  • üî¥ ROUGE ‚Äî checks how much of the reference is ‚Äúcovered‚Äù in the generated output.
  • üîµ BLEU ‚Äî checks how many word patterns from the reference appear in the output.
  • üü¢ METEOR ‚Äî rewards matching important words, including synonyms, and considers word order.

Let‚Äôs understand each one with:

  • Simple real-life examples
  • Easy math formulas
  • Python code with explanations

üî¥ 1. ROUGE (Recall-Oriented Understudy for Gisting Evaluation)

üß† What is ROUGE?

ROUGE checks how much of the reference text is captured by the generated text.

It‚Äôs mainly used to evaluate summarization tasks.

ROUGE comes in types like:

  • ROUGE-1: matches on individual words
  • ROUGE-2: matches on word pairs (bigrams)
  • ROUGE-L: matches based on longest common sequence of words

‚úèÔ∏è Simple Formula

Let‚Äôs say:

  • Ref_words = number of words in the reference
  • Match_words = number of matching words between candidate and reference

Then:

ROUGE Recall = Match_words / Ref_words

You can also calculate precision and F1-score:

Precision = Match_words / Candidate_words
F1 = (2 * Precision * Recall) / (Precision + Recall)

üß™ Example

Reference Summary:
"The cat is sleeping on the mat"

Model Output (Candidate):
"The cat is on the mat"

Step-by-step:

  • Reference words: 7
  • Candidate words: 6
  • Matching words: "The", "cat", "is", "on", "the", "mat" ‚Üí 6 matches

So:

  • Recall = 6 / 7 ‚âà 0.857
  • Precision = 6 / 6 = 1.0
  • F1 = (2 √ó 0.857 √ó 1.0) / (0.857 + 1.0) ‚âà 0.923

üêç Python Code for ROUGE-1

def rouge_1_score(reference, candidate):
    ref_words = reference.lower().split()
    cand_words = candidate.lower().split()
    
    # Count matches
    matches = sum(1 for word in cand_words if word in ref_words)
    
    recall = matches / len(ref_words)
    precision = matches / len(cand_words)
    f1 = 2 * precision * recall / (precision + recall)
    
    return round(precision, 2), round(recall, 2), round(f1, 2)

# Example
ref = "The cat is sleeping on the mat"
cand = "The cat is on the mat"
print("ROUGE-1:", rouge_1_score(ref, cand))

üß† Code Explained

  • lower().split() ‚Üí converts sentence to lowercase and splits into words
  • sum(...) ‚Üí counts how many words in candidate are found in reference
  • The rest calculates the standard precision, recall, and F1 formulas.

üîµ 2. BLEU (Bilingual Evaluation Understudy)

üß† What is BLEU?

BLEU checks how many word patterns (1-word, 2-word, 3-word‚Ä¶) from the reference are reused by the model.

It‚Äôs widely used for machine translation.

Unlike ROUGE, BLEU focuses more on precision (how accurate the output is) and applies a penalty for short outputs.

‚úèÔ∏è Simple Formula

Let‚Äôs say:

  • Precision_n = fraction of n-grams (like 2-word phrases) in the candidate that match reference
  • BP = brevity penalty for short output

Then:

BLEU = BP √ó average of Precision_1 to Precision_4

üß™ Example

Reference: "The cat is on the mat"
Candidate: "The cat is on the floor"

Matching bigrams (2-grams):

  • Reference: ["The cat", "cat is", "is on", "on the", "the mat"]
  • Candidate: ["The cat", "cat is", "is on", "on the", "the floor"]

Matched 4 out of 5 bigrams
‚Üí Precision = 4 / 5 = 0.8
‚Üí Lengths are same, so no brevity penalty
‚Üí BLEU = 0.8

üêç Python Code for BLEU-2 (2-word phrases)

from collections import Counter
def bleu_2_score(reference, candidate):
    def get_bigrams(words):
        return [tuple(words[i:i+2]) for i in range(len(words)-1)]
    ref_words = reference.lower().split()
    cand_words = candidate.lower().split()
    ref_bigrams = Counter(get_bigrams(ref_words))
    cand_bigrams = Counter(get_bigrams(cand_words))
    
    # Count overlapping bigrams
    match = sum(min(cand_bigrams[bg], ref_bigrams.get(bg, 0)) for bg in cand_bigrams)
    
    precision = match / max(1, sum(cand_bigrams.values()))
    return round(precision, 2)
# Example
ref = "The cat is on the mat"
cand = "The cat is on the floor"
print("BLEU-2:", bleu_2_score(ref, cand))

üß† Code Explained

  • get_bigrams() ‚Üí makes 2-word combinations from a sentence
  • Counter() ‚Üí counts how many times each bigram occurs
  • min(..., ...) ‚Üí avoids over-counting repeated bigrams
  • Final result = % of bigrams in candidate that also appear in reference

üü¢ 3. METEOR (Metric for Evaluation of Translation with Explicit ORdering)

üß† What is METEOR?

METEOR gives credit for synonyms, word stems, and checks if the order of words is correct.

It‚Äôs more accurate than BLEU and ROUGE when the model uses different words but has the same meaning.

‚úèÔ∏è Simple Formula

Let‚Äôs say:

  • M = number of matching words (including synonyms/stems)
  • P = precision = M / candidate length
  • R = recall = M / reference length

Then:

F = (10 * P * R) / (R + 9P)
Penalty = 0.5 * (chunks / M)^3
METEOR = F * (1 - Penalty)

Where chunks = how many groups of words are in the same order

üß™ Example

Reference: "The president gave a speech to the nation"
Candidate: "The leader gave a talk to the country"

Let‚Äôs assume:

  • "president" ‚âà "leader" (synonym)
  • "speech" ‚âà "talk"
  • "nation" ‚âà "country"

So:

  • M = 6 matches
  • P = 6 / 7
  • R = 6 / 7
  • Chunk count = 2 (words appear mostly in same order)

Use formula to compute METEOR.

üêç Python Code (basic version without synonym support)

def meteor_score(reference, candidate):
    ref_words = reference.lower().split()
    cand_words = candidate.lower().split()
    matches = [(i, ref_words.index(word)) for i, word in enumerate(cand_words) if word in ref_words]
    M = len(matches)
    if M == 0:
        return 0.0
    P = M / len(cand_words)
    R = M / len(ref_words)
    F = (10 * P * R) / (R + 9 * P)
    # Count chunks (words in same order)
    chunks = 1
    for k in range(1, M):
        if matches[k][1] != matches[k-1][1] + 1:
            chunks += 1
    penalty = 0.5 * (chunks / M) ** 3
    score = F * (1 - penalty)
    return round(score, 2)
# Example
ref = "The president gave a speech to the nation"
cand = "The president gave a talk to the country"
print("METEOR:", meteor_score(ref, cand))

üß† Code Explained

  • enumerate(cand_words) ‚Üí loop through candidate words
  • word in ref_words ‚Üí simple exact match
  • matches ‚Üí keeps track of matched positions
  • chunks ‚Üí increases when matched words are out of order
  • Final METEOR = F-score √ó (1 ‚Äî penalty for disorder)

‚úÖ Final Notes

  • Use ROUGE when checking if a summary covers all important info.
  • Use BLEU when accuracy of translation matters.
  • Use METEOR when semantic meaning and word order are important.

üîÅ Try combining all three to get a balanced view of LLM output quality.
